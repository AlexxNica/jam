Dashboard Tabs

When we want to summarize a tab's content, we use a control that we're calling .tabBoard (Tabs + Dashboard). It works. This is used typically within reporting contexts where users can learn all the important information at a glance, but then select the stat to view a further detailed breakdown.

NOTE: There should be no more than three tabs per a group.

Variant Styles

Condensed Table

The condensed table does pretty much what it sounds like it would do: condenses the padding and line-height so that the table has a tighter feeling to it. This is suggested for data tables such as found within reports that have multiple records.

.table._condensed

When tables have multiple columns, it is suggested to add row borders to help the user more easily track the table data. This is suggested for data tables such as found within reports.

Table Sizing

While the browser will prescribe column sizes based on content, sometimes we may want table columns to adhere to a specific width. To do this, we leverage the same class names used within the grid.

When applying a column size, it is only necessary to apply it within the first table cell within a column as the rest of the column will abide by the width its smallest cell.

NOTE: Remember the overall column total per row cannot exceed 12 columns. Otherwise this will cause your table row to wrap.

._textLeft, ._textCenter, ._textRight
