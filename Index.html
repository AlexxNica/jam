Typography

Typography is an important design element for Stack Overflow Careers. Strong typography applied consistently can not only build brand recognition and trust, but it can also help establish reliable communication patterns for our end users. To accomplish this goal, we have selected the font Merriweather Sans ST. This font was designed specifically for on-screen use, allowing for greater control of line-heights and letter-spacing natively within our CSS without resorting to Javascript.

Font Weight

With the addition of webfonts, font-weight plays a greater role than it once did. Previously a "bold" weight was programmatically determined using an algorithm to fatten a font. Webfonts however are hinted specifically for declared weight values. Stack Overflow Careers uses a font-weight numerical value for the best font representation. These have been abstracted into five different variables:

//  Font Weight Variables
@@fontWeight-hairline:       100;
@@fontWeight-extraLight:     200;
@@fontWeight-light:          300;
@@fontWeight-book:           400;
@@fontWeight-extraBold:      700;

To better manage use of font-weights within Stack Overflow Careers, raw font-weights should not be entered within CSS. Instead use the following mixin to declare the desired weight:

//  Font Weight Mixin Guard to quickly assign font-weight variables
.fontWeight (@@w) {
  & when (@@w = 1) { font-weight: @@fontWeight-hairline; }
  & when (@@w = 2) { font-weight: @@fontWeight-extraLight; }
  & when (@@w = 3) { font-weight: @@fontWeight-light; }
  & when (@@w = 4) { font-weight: @@fontWeight-book; }
  & when (@@w = 7) { font-weight: @@fontWeight-extraBold; }
}

To declare a font-weight, pass your desired weight to the mixin:

h1 { .fontWeight(1); }

This will output:

h1 { font-weight: 100; }

Line Heights

Do not declare values with line-heights (i.e. px, em). Keeping them unitless allows the value to be relative to the declared font-size, which provides greater flexibility. Instead of trying to rememeber a rather long number like 1.428571429 or variations thereof to create tighter or looser line heights, a number of variables have been created.

//  Line Height Variables
@@lineHeight-tightest:       (@@fontSize-base + 2) / @@fontSize-base;
@@lineHeight-tighter:        (@@fontSize-base + 4) / @@fontSize-base;
@@lineHeight-tight:          (@@fontSize-base + 6) / @@fontSize-base;
@@lineHeight-base:           @@fontSize-base / @@fontSize-base;
@@lineHeight-loose:          (@@fontSize-base + 8) / @@fontSize-base;
@@lineHeight-looser:         (@@fontSize-base + 10) / @@fontSize-base;
@@lineHeight-loosest:        (@@fontSize-base / @@fontSize-base) * 2;

To make reduce the burden in declaring line-heights, a mixin guard has been created. Now all one has to do is pass along how loose or tight they desire the lines to be and the appropriate line-height variable is inserted. The mixin guard looks like this:

//  Line Height Mixin Guard to quickly assign line-height variables
.lineHeight (@@lh) {
  & when (@@lh = base)     { line-height: @@lineHeight-base; }
  & when (@@lh = tight)    { line-height: @@lineHeight-tight; }
  & when (@@lh = tighter)  { line-height: @@lineHeight-tighter; }
  & when (@@lh = tightest) { line-height: @@lineHeight-tightest; }
  & when (@@lh = loose)    { line-height: @@lineHeight-loose; }
  & when (@@lh = looser)   { line-height: @@lineHeight-looser; }
  & when (@@lh = loosest)  { line-height: @@lineHeight-loosest; }
}

To declare a line-height, pass your desired height into the mixins:

h1 { .lineHeight(loose); }

This will output:

h1 { line-height: 1.571428571; }
