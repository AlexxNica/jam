Grid System

Introduction

This kit includes a responsive, fluid grid system that scales up to 12 columns as the device or viewport size increases. It includes predefined classes for easy layout options, as well as mixins for generating more semantic layouts.

The grid system that we chose is based off of Bootstrap's 12 column grid. If you have used it before, the conventions here are nearly identical, with the exception of some class naming conventions.

  • Rows must be placed within a .container for proper alignment. There is usually one per page and it contains the contents of that page.
  • Use rows to create horizontal groups of columns.
  • Content should be placed within columns, and only columns may be immediate children of rows.
  • Predefined grid classes like .-row and .-col4 are available for quickly making grid layouts. Less mixins can also be used for more semantic layouts.
  • Unlike Bootstrap, columns, rows, and containers do not have built in padding or margins.
  • Grid columns are created by specifying the number of twelve available columns you wish to span. For example, three equal columns would use three .-col4.
  • If more than 12 columns are placed within a single row, each group of extra columns will, as one unit, wrap onto a new line.
  • Unlike Bootstrap, we do not allow column restructuring based on device or viewport size. In other words, we do not allow for a row to contain .-col6 on two divs for a tablet, but allow for switching to one div being .-col3 and the other .-col9 on a mobile device.

Less Mixins & Variables

We use the following media queries in our Less files to create the key breakpoints in our grid system.

                            @media (min-width: @mediaQuery-small)  { max-width: @mediaQuery-small - 18; }
                            @media (min-width: @mediaQuery-medium) { max-width: @mediaQuery-medium - 22; }
                            @media (min-width: @mediaQuery-large)  { max-width: @mediaQuery-large - 30; }

The variables used to defined these widths are as follows

                            @mediaQuery-small:      768px;
                            @mediaQuery-medium:     992px;
                            @mediaQuery-large:      1200px;

Typography

Introduction

Typography is an important design element for Stack Overflow Careers. Strong typography applied consistently can not only build brand recognition and trust, but it can also help establish reliable communication patterns.

Our general approach to typography is to apply styles to raw html tags including p, h*, strong, and em tags so it stays consistent across the project. This even extends to multi-tag elements like ul, li, blockquote, and dd. In addition, we provide specific use classes like ._hugeTitle and ._pageTitle that are meant for particular parts of the page.

PLEASE NOTE: If you are altering pre-determined font-families, font-sizes, font-weights, line-heights, or letter-spacing, please read the type LESS documentation.

Typefaces

To communicate a professional, yet friendly personality, Stack Overflow Careers has selected the typeface Merriweather Sans ST. This font was designed specifically for on-screen use, allowing for greater control of line-heights and letter-spacing natively within our CSS without resorting to Javascript. While we may use other weights occasionally, most copy will use Light, Normal, and Bold styles. The following are examples of how they would appear on large type.

Headlines

Headlines provide signposts for content as well express content priority. Headlines should descend in order. This means that a <h3> shouldn't precede a <h1> in the DOM order. The question to ask yourself is, "if this page's CSS didn't load, would this copy still make sense?"

This is a subheading

This is the wrong thing to do

Out-of-Order Headings

Header tags communicate content order such as titles, section titles and sub-section titles. A header tag should not be chosen based on the desired style, but on where it is located within the content. Different headline styles can be applied two different ways:

  1. Apply a header class such as .h3 to apply another header's base style to a headline.
  2. Use a specific style class such as .text ._large to change a header's appearance.

In the rare case that you would like the visual aesthetics you would get by combining a heading hierarchy out of order, you could do the following instead.
